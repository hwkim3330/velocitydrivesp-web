User Interface

Introduction

VelocityDRIVE CT is powered by a feature rich user interface. For information on using the software's command line interface, please see Chapter 6: “Command Line Tool Interface”.

The major functions of the software's graphic user interface are organized within the top ribbon of the application in a row of tabs This chapter outlines the functions and navigation of each tab.

Devices and Configurations

Before interacting with the user interface, it is important to understand the underlying VelocityDRIVE CT workflow.

Changes are not made directly to a device’s YANG instances/datastores, but are instead performed first in a configuration within the Configurations Tab. A configuration is a set of YANG instances retrieved from the device that are allowed to be modified by the user. Once retrieved, the user is free to make changes to the instances contained in the configuration. These changes do not yet affect the switch directly. Once the user is satisfied with their changes, the revised values in the configuration can uploaded to the device temporarily by performing a sync. If successful, the tool will re-read the running datastore of the device. The changed values (and affected YANG instances) can then be evaluated from the Devices Tab.

Explained in a different way, think of the YANG instances displayed in the Devices Tab as the current state of the switch (the running datastore). The YANG instances shown in the Configurations Tab is a possible state of the switch.

To better understand the difference, the following sequence outlines the basic operating process of VelocityDRIVE CT.

  1. The user connects a valid switch to their PC and opens the VelocityDRIVE CT software. The software will then recognize the device.
  2. Once the device is identified, VelocityDRIVE CT downloads the device's corresponding YANG/CORECONF library.
  3. The library is loaded into the Devices Tab, which is then populated by actual data from the connected switch. This is the current state of the switch.
  4. Once the YANG instances are retrieved, they may be edited in the Configurations Tab. A configuration, which can either be a patch to specific YANG instances or a whole datastore, can be thought of as a possible state of the entire switch or some specific part of its configuration.
  5. Changes made in a configuration will not be seen on the switch until the configuration is applied to the running datastore. Whole datastore changes can be saved permanently in the device’s startup memory (only seen on the device after a reboot command or a push of the reset button). Patches made to the running datastore are temporary. Patches can be saved permanently, but only as a part of a whole datastore in the device’s startup memory.

Navigation Buttons

A pair of Navigation buttons can be found in the top left corner of the VelocityDRIVE™ CT user interface ribbon. These backwards and forward navigators can be used to quickly return to previous tabs, screens, and tree structure nodes throughout the software.

Project Tab

VelocityDRIVE CT utilizes Project files to store data from Configurations and Devices. Fillable fields within the main Project tab are available to annotate an individual Project file's relevant properties and information.

Project Menu Definitions

Figure 3-1: VelocityDRIVE CT Project File Menu

The Project menu is accessed by clicking on the Menu button , presenting the following options:

  • New Project: Creates a new project file with the default name of “New Project.” It is recommended to change the default project name to a unique identifier.
  • Open Project: Selects and loads a previously saved Project file.
  • Save: Saves the current Project file.
  • Save As: Prompts for a new project file name.

Project Files

A saved VelocityDRIVE CT Project file contains:

  • All Configurations created within the Project
  • Remembered Device connection information, which includes:
    • Device connection settings (URI)
    • Last connection settings
    • Device's YANG Catalog ID Checksum

YANG Library Tab

When VelocityDRIVE CT recognizes a connected device operating with VelocityDRIVE SP compatible firmware, it automatically loads the corresponding YANG library in the YANG Library tab.

YANG Library Tab Menu

YANG Library Tab Menu

The YANG Library tab displays each YANG library as a separate card. The card contains:

  • YANG Library Checksum ID: The checksum ID is a unique identifier for the YANG library associated with that device's specific hardware and firmware combination.
  • Recent Connections: A list of recently connected devices that are associated with that particular YANG library.
  • Refresh button : If a currently connected device is not present but should match this YANG library in the Project, the Refresh button can be used to confirm the Device is present and matches the checksum.
  • Open button : Opens that YANG library's element catalog

YANG Library Catalog

YANG Library Catalog

Opening the library from the YANG Library tab to reveals the list of YANG modules. The catalog is displayed with the following columns of information:

  1. The YANG Library checksum ID that corresponds to the Device and Firmware
  2. The Name of the YANG Library module.
  3. The Namespace address in the YANG library tree in which the element can be found.
  4. The element’s Prefix value, which acts as a shortcut reference to the module.
  5. The Organization responsible for maintaining the YANG module’s characteristics.
  6. That last time the responsible Organization made a Revision to the standards for this particular element.

Note: The values found in items 3-6 are defined by the YANG module itself

Clicking on the Name of a YANG library element will display its code

Configurations Tab

The VelocityDRIVE CT Configurations tab displays cards of all Configurations contained within a Project file.

Configurations Tab Menu

Configurations Tab Menu

A Configuration represents a whole or partial datastore instance of a YANG library. Each Configuration is saved within its own card, which is labeled with a user defined Name and Description.

  • When VelocityDRIVE CT is first launched, or a new Project file is created (Section 3.4.1 “Project Menu Definitions”), a blank Configuration card named “Default” is generated. Inputing a new Configuration name is described in Section 4.3.3 “Naming VelocityDRIVE CT Configurations”.
  • Additional empty Configurations can be created by using the Add button .
  • Configurations may be removed by using the Delete button .
  • The Open button brings up the Configurations tab YANG Catalog interface.

Log Tab

Figure 3-12: Log View

In Figure 3-12, we see the Log tab main screen view. It is broken up into four sections.

  1. The main log entry view provides a comprehensive list of all log events. Log events are ordered chronologically in descending order from oldest to newest. Each log event displays information in the following columns

    • Index: the index column shows the individual index number for each log event
    • Time: the system time of the log event.
    • Severity: indicates if the event is conveying information (INFO) or if the event is a possible error (WARNING)
    • Type: this column categorizes Log event. A Trace type provides information on VelocityDRIVE CT itself, a Device type provides information on the connected switch, a Send type indicates the Log event is a message from the tool to the device, and a Receive type is a message received by the tool from the network or device.
    • Source: this column shows the origin location of the Log event.
    • Protocol: this column lists the format in which the Log event was sent or received.
    • Length: this column provides the byte size of the data that was transferred
    • Description: this column lists a description of the nature of the Log event

  2. The Log Details panel provides expanded information regarding the Log event. In addition to the information from the columns in the main event list, it also displays information on the Status of the message (Segmented, Not Segmented, or Not Applicable), as well as the content of the message in CBOR or Binary formats (where available). The details panel also features a Filters tab. More details on the Filters tab is available in Section 3.8.0.1 “Filters Functionality”

  3. The lower Log panel features several useful functions

    • This icon indicates that the log feed is in Auto Scroll mode. Auto Scroll allows for new log messages to populate at the bottom of the main panel in real time. Pressing the button toggles between Auto Scroll and Analysis modes.
    • This icon represents Analysis mode. In this mode, auto scrolling is paused, allowing the user to select a single event without the screen moving on its own.
    • Use this button to locate the log file location of the selected Log event.
    • Index of the Message and Go: Use this box and the “Go” button to jump to the location of a specific Event index number.

  4. Using this panel, Log files from previous sessions can be loaded and viewed. Use the Open button to open a file saved elsewhere on your machine or from the default Log file location. In Windows, log files are located here: 

                                C:\Users\{{USERNAME}}\AppData\Roaming\microchip-velocity-
                                drivect\logs.

    You may return to the live Log view by pressing the drop down carrot and choose “Online as source.

Search Tool

A search tool is provided to aide in finding relevant variables and values.

Figure 3-14: Search Bar

As seen in Figure 3-14, the Search tool is available in the top right of the navigation ribbon. It can be used while in any other tab, and its results appear as an overlay on top of the display currently in view.

Search results appear in a right-side panel displaying results by origin: Configurations or Devices. The search results panel also includes a filter for Configurations and Devices, allowing you to limit results to a single domain.

Figure 3-15: Search Results

Each result displays the name of the data item, as well as information about its origin (Configuration name if this is a result value, connection ID if this is a Device value). The full instance ID is located under the name of the retrieved data item, allowing users to distinguish between similarly named results. Clicking on a result will take you to the page containing the item in question.

Search terms are case insensitive.

Note: The Search results panel is collapsible by using the < button to the left of Results.
